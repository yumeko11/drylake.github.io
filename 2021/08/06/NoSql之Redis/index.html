NoSql之Redis

一、Redis简介

1、什么是Redis

Redis是现在最受欢迎的NoSQL数据库之一 使用ANSI C编写的开源、包含多种数据结构、支持网络、基于内存、可选持久性的键值对存储数据库

2、Redis的特性

  • 基于内存运行 性能高效
  • 支持分布式 理论上可以无限扩展
  • key-value存储系统
  • 开源的使用ANSI C语言编写、遵守BSD协议、支持网络、可基于内存亦可持久化的日志型数据库、提供多种语言的API

3、Redis的优点

  • C/S通讯模型
  • 单进程单线程模型
  • 丰富的数据类型
  • 操作具有原子性
  • 持久化
  • 高并发读写
  • 支持lua脚本

4、Redis的应用场景

缓存系统(“热点”数据：高频读、低频写)、计数器、消息队列系统、排行榜、社交网络和实时系统

二、Redis的安装

1、在Linux系统安装

  1. 下载redis

    https://redis.io/

  2. 将压缩文件复制到opt文件夹下
    • cd /opt
    • tar -zxvf redis-6.2.5.tar.gz解压文件
    • cd redis-6.2.5
    • yum install gcc下载C语言编译
    • make编译
    • make install安装
  3. 新建文件夹将redis.conf文件备份
    • mkdir /opt/myRedis
    • cp redis.conf /opt/myRedis/redis.conf
    • cd /opt/myRedis/
    • vi redis.conf
    • 将配置文件下的daemonize no改成daemonize yes 将其设置为后台启动
  4. 启动redis服务
    • cd /opt/myRedis/
    • redis-server redis.conf
  5. 启动redis客户端
    • redis-cli -h -p
  6. 重启redis
    1. 客户端重启
      • shutdown
      • redis-cli -p 6379 shutdown
      • redis-server redis.conf
    2. 杀进程重启
      • ps -ef | grep redis查看进程
      • kill 进程号
      • redis-server redis.conf

三、Redis数据结构及常用语法

1、String(字符串)

Redis 字符串数据类型的相关命令用于管理redis字符串值。

命令 描述 实例
set key value 设置指定key的值 set k1 v1
get key 获取指定key的值 GET k1
getrange key start end 返回key中字符串值的子字符 getrange k1 0 -1
getset key value 将给定key的值设为value,并返回 key 的旧值 getset k1 v2
mget key1 key2.. 获取所有(一个或多个)给定key的值 mget k1 k2 k3
setex key seconds value 将值value关联到key,并将key的过期时间设为seconds(以秒为单位) setex k1 60 v1
psetex key milliseconds value 将值value关联到key,并将key的过期时间设为milliseconds(以毫秒为单位) setex k1 1000 v1
setnx key value 只有在key不存在时设置key的值 setnx k2 v2
strlen key 返回key所储存的字符串值的长度 strlen k1
mset key1 value1 key2 value2… 同时设置一个或多个key-value对 mset k1 v1 k2 v2
msetnx key1 value1 key2 value2… 同时设置一个或多个key-value对,当且仅当所有给定key都不存在 msetnx k1 v1 k2 v2
incr key 将key中储存的数字值增一 incr k1
decr key 将key中储存的数字值减一 decr k1
incrby key increment 将key所储存的值加上给定的增量值(increment) incrby key 10
decrby key decrement 将key所储存的值加上给定的减量值(decrement) decrby key 10
append key value 如果key已经存在并且是一个字符串,append命令将指定的value追加到该key原来值(value)的末尾 append k1 hi

2、Hash(哈希)

Redis hash是一个string类型的field(字段)和value(值)的映射表,hash特别适合用于存储对象。

命令 描述 实例
hdel key field1 field2… 删除一个或多个哈希表字段 hdel k1 f1 f2
hexists key field 查看哈希表key中,指定的字段是否存在 hexists k1 f1
hget key field 获取存储在哈希表中指定字段的值 hget k1 f1
hgetall key 获取在哈希表中指定key的所有字段和值 hgetall k1
hincrby key field increment 为哈希表key中的指定字段的整数值加上增量increment hincrby k1 f1 10
hkeys key 获取所有哈希表中的字段 hkeys k1
hlen key 获取哈希表中字段的数量 hlen k1
hmget key field1 field2… 获取所有给定字段的值 hmget k1 f1 f2
hmset key field1 value1 field2 value2… 同时将多个field-value(域-值)对设置到哈希表key中 hmset k1 f1 v1 f2 v2
hset key field value 将哈希表key中的字段field的值设为value hset k1 f1 v1
hsetnx key field value 只有在字段field不存在时,设置哈希表字段的值 hsetnx k1 f1 v1
hvals key 获取哈希表中所有值 hvals k1

3、List(列表)

Redis列表是简单的字符串列表,按照插入顺序排序。你可以添加一个元素到列表的头部(左边)或者尾部(右边)。

命令 描述 实例
lindex key index 通过索引获取列表中的元素 lindex k1 0
linsert key before/after pivot value 在列表的元素前或者后插入元素 linsert k1 before/after hi1 hi
llen key 获取列表长度 llen k1
lpop key 移出并获取列表的第一个元素 lpop k1
lpush key value1 value2… 将一个或多个值插入到列表头部 lpush k1 v1 v2
lpushx key value 将一个值插入到已存在的列表头部 lpushx k1 v1
lrange key start stop 获取列表指定范围内的元素 lrange k1 0 -1
lrem key count value 移除列表元素 lrem k1 -2 value
lset key index value 通过索引设置列表元素的值 lset k1 3 value
ltrim key start stop 对一个列表进行修剪,让列表只保留指定区间内的元素 ltrim k1 2 4
rpop key 移除列表的最后一个元素,返回值为移除的元素 rpop k1
rpush key value1 value2… 在列表中添加一个或多个值 rpush k1 v1 v2
rpushx key value 为已存在的列表添加值 rpushx k1 v1 v2

4、Set(集合)

Redis的Set是String类型的无序集合。集合成员是唯一的,这就意味着集合中不能出现重复的数据。
集合对象的编码可以是intset或者hashtable。

命令 描述 实例
sadd key member1 member2… 向集合添加一个或多个成员 sadd k1 m1 m2
scard key 获取集合的成员数 scard k1
sdiff key1 key2… 返回第一个集合与其他集合之间的差异 sdiff k1 k2
sdiffstore destination key1 key2… 返回给定所有集合的差集并存储在destination中 sdiffstore destination k1 k2
sinter key1 key2… 返回给定所有集合的交集 sinter k1 k2
sinterstore destination key1 key2… 返回给定所有集合的交集并存储在destination中 sinterstore destination k1 k2
sismember key member 判断member元素是否是集合key的成员 sismember k1 m1
smembers key 返回集合中的所有成员 smembers k1
smove source destination member 将member元素从source集合移动到destination集合 smove source destination m1
spop key 移除并返回集合中的一个随机元素 spop k1
srandmember key count 返回集合中一个或多个随机数 srandmember k1 2
srem key member1 member2… 移除集合中一个或多个成员 srem k1 m1 m2
sunion key1 key2… 返回所有给定集合的并集 sunion k1 k2
sunionstore destination key1 key2… 所有给定集合的并集存储在destination集合中 sunionstore destination k1 k2

5、zset(sorted set:有序集合)

Redis有序集合和集合一样也是 string 类型元素的集合,且不允许重复的成员。
不同的是每个元素都会关联一个 double 类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。
有序集合的成员是唯一的,但分数(score)却可以重复。

命令 描述 实例
zadd key score1 member1 score2 member2… 向有序集合添加一个或多个成员,或者更新已存在成员的分数 zadd k1 s1 m1 s2 m2
zcard key 获取有序集合的成员数 zcard k1
zcount key min max 计算在有序集合中指定区间分数的成员数 zcount k1 10 20
zincrby key increment member 有序集合中对指定成员的分数加上增量increment zincrby k1 5 m1
zlexcount key min max 在有序集合中计算指定字典区间内成员数量 zlexcount k1 5 15
zrange key start stop withscores 通过索引区间返回有序集合指定区间内的成员 zrange k1 1 10 withscores
zrank key member 返回有序集合中指定成员的索引 zrank k1 m1
zrem key member member2… 移除有序集合中的一个或多个成员 zrem k1 m1 m2
zremrangebylex key min max 移除有序集合中给定的字典区间的所有成员 zremrangebylex k1 0 -1
zremrangebyrank key start stop 移除有序集合中给定的排名区间的所有成员 zremrangebyrank k1 1 10
zremrangebyscore key min max 移除有序集合中给定的分数区间的所有成员 zremrangebyscore k1 10 20
zscore key member 返回有序集中,成员的分数值 zscore k1 m1
zrevrank key member 回有序集合中指定成员的排名,有序集成员按分数值递减(从大到小)排序 zrevrank k1 m1
